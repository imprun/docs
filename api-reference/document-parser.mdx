<Warning>
  이 API는 **베타** 단계입니다. 파서가 안정화되기 전까지 응답 스키마가 변경될 수 있습니다.
  연동 시 `parser_version` 필드를 확인하여 버전별 분기 처리를 권장합니다.
</Warning>

등기부등본(등기사항전부증명서) PDF 파일을 업로드하면 **표제부·갑구·을구** 전체를 구조화된 JSON으로 변환합니다.

## 엔드포인트

```
POST /document-parser/registry
```

문서 유형(토지/건물/집합건물)은 PDF 내용에서 자동 판별됩니다. 별도 지정이 필요 없습니다.

### 요청

<CodeGroup>
```bash Production
curl -X POST https://prod.fn.imprun.dev/document-parser/registry \
  -H "Authorization: Bearer {keyId}.{secret}" \
  -F "file=@등기부등본.pdf"
```

```bash Staging
curl -X POST https://staging.fn.imprun.dev/document-parser/registry \
  -H "Authorization: Bearer {keyId}.{secret}" \
  -F "file=@등기부등본.pdf"
```
</CodeGroup>

| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|:----:|------|
| `file` | file | O | 등기부등본 PDF 파일 (최대 50MB) |

### 응답

```json
{
  "document_type": "registry",
  "parser_version": "1.5.0",
  "parsed_at": "2026-02-25T12:00:00+00:00",
  "data": { ... },
  "confidence": {
    "overall": 91,
    "level": "높음",
    "details": { ... }
  }
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `document_type` | `string` | 항상 `"registry"` |
| `parser_version` | `string` | 파서 버전. 현재 `"1.4.0"` (운영) 또는 `"1.5.0"` (스테이징) |
| `parsed_at` | `string` | 파싱 수행 시각 (ISO 8601) |
| `data` | `object` | 파싱 결과 본문. 버전에 따라 구조가 다름 — 아래 참조 |
| `confidence` | `object` | 파싱 신뢰도 점수 |

<Note>
  현재 Production은 v1.4.0, Staging은 v1.5.0이 배포되어 있습니다.
  v1.5.0이 안정화되면 Production에도 적용됩니다.
  이 문서는 **v1.5.0 스키마**를 기준으로 작성되었습니다.
</Note>

### 에러

| HTTP 코드 | 에러 | 설명 |
|-----------|------|------|
| `400` | 잘못된 요청 | `file` 파라미터 누락, 지원하지 않는 문서 타입 |
| `401` | 인증 실패 | API 키가 없거나 유효하지 않음 |
| `403` | 접근 거부 | 잔액 부족 또는 계정 정지 |
| `422` | 파싱 실패 | PDF에서 등기부등본 내용을 인식할 수 없음 |

## RegistryDocument

`data` 필드에 담기는 v1.5.0 최상위 구조입니다.

```json
{
  "meta": {
    "parser_version": "1.5.0",
    "parsed_at": "2026-02-25T12:00:00+00:00",
    "document_id": null
  },
  "profile": {
    "certificate_type": "full",
    "include_cancellations": true,
    "property_type": "land",
    "layout_profile": "registry_v1_standard"
  },
  "source": {
    "page_count": 5,
    "text_length": 8240,
    "pdf_text_based": true
  },
  "unique_number": "1184-2004-006616",
  "property_address": "전라남도 나주시 빛가람동 42-2",
  "jurisdiction": "광주지방법원 나주등기소",
  "viewed_at": "2025년 04월 01일 13시 06분 16초",
  "issued_at": null,
  "title": { ... },
  "section_a": [ ... ],
  "section_b": [ ... ],
  "trade_lists": [ ... ],
  "major_summary": { ... },
  "diagnostics": { ... },
  "raw_text": ""
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `meta` | `object` | [DocumentMeta](#documentmeta) — 파서 메타데이터 |
| `profile` | `object` | [DocumentProfile](#documentprofile) — 문서 프로파일 |
| `source` | `object` | [SourceInfo](#sourceinfo) — 입력 소스 정보 |
| `unique_number` | `string` | 고유번호 (`"NNNN-YYYY-NNNNNN"`) |
| `property_address` | `string` | 부동산 소재지 주소 |
| `jurisdiction` | `string \| null` | 관할등기소 |
| `viewed_at` | `string \| null` | 열람일시 |
| `issued_at` | `string \| null` | 발행일시 |
| `verification_image` | `string \| null` | 검증 바코드 이미지 (Base64) |
| `title` | `object` | [표제부](#표제부-title) — 부동산 물리 정보 |
| `section_a` | `array` | [갑구](#갑구을구-section_asection_b) — 소유권에 관한 사항 |
| `section_b` | `array` | [을구](#갑구을구-section_asection_b) — 소유권 이외의 권리 |
| `trade_lists` | `array` | [매매목록](#매매목록) — 토지 복수 필지 거래 시 |
| `major_summary` | `object \| null` | [주요등기사항 요약](#주요등기사항-요약) |
| `diagnostics` | `object` | [ParseDiagnostics](#parsediagnostics) — 진단 정보 |
| `raw_text` | `string` | 원본 전체 텍스트 (디버깅용) |

<Note>
  `profile.property_type`에 따라 `title` 내부 필드가 달라집니다. 유형별 상세 스키마는 각 문서 유형 페이지를 참고하세요.
</Note>

## DocumentMeta

파서 버전과 파싱 시각 정보입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `parser_version` | `string` | 파서 버전 (예: `"1.5.0"`) |
| `parsed_at` | `string` | 파싱 수행 시각 (ISO 8601) |
| `document_id` | `string \| null` | PDF SHA256 해시 (선택) |

## DocumentProfile

문서의 유형과 특성을 나타냅니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `certificate_type` | `string` | `"full"` (등기사항전부증명서) \| `"partial"` (일부증명서) |
| `include_cancellations` | `boolean` | 말소사항 포함 여부 |
| `property_type` | `string` | `"land"` \| `"building"` \| `"aggregate_building"` |
| `layout_profile` | `string` | 레이아웃 프로파일 (예: `"registry_v1_standard"`) |

## SourceInfo

입력 PDF의 물리적 정보입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `page_count` | `integer` | PDF 페이지 수 |
| `text_length` | `integer` | 추출된 텍스트 총 문자 수 |
| `pdf_text_based` | `boolean` | 텍스트 기반 PDF 여부 (`false`면 스캔 이미지) |

## 갑구·을구 (section_a/section_b)

갑구는 소유권에 관한 사항, 을구는 소유권 이외의 권리(근저당권, 전세권, 임차권, 지상권 등)를 포함합니다.
v1.5.0에서는 갑구와 을구가 동일한 `RegistryEntry` 구조를 사용합니다.

```json
{
  "entry_id": "gap:p2:r5",
  "rank_number": "3",
  "previous_rank": null,
  "purpose": {
    "raw": "소유권이전",
    "normalized": "소유권이전",
    "right_type": "ownership_transfer",
    "action_type": "create",
    "target_rank": null,
    "target_type": null,
    "target_owner": null,
    "sub_purposes": []
  },
  "receipt": {
    "date": "2007년 09월 11일",
    "number": "14543호"
  },
  "cause": {
    "text": "매매",
    "date": "2007년 08월 30일"
  },
  "content": {
    "persons": [
      {
        "name": "홍길동",
        "role": "소유자",
        "resident_number": "800101-*******",
        "address": "서울특별시 강남구 역삼동 123-4",
        "share": null
      }
    ],
    "amounts": [],
    "shares": [],

    "term": null,
    "lease": null,

    "prohibition_text": null,

  "status": {
    "is_cancelled": false,

    "page_no": 2,
    "bbox": null,

    "is_visual_cancelled": false

  "raw_text": "...",
  "confidence": 0.95

```

### RegistryEntry

| 필드 | 타입 | 설명 |
|------|------|------|
| `entry_id` | `string` | 고유 식별자. 갑구: `"gap:p2:r5"`, 을구: `"eul:p3:r8"`. 형식: `{섹션}:p{페이지}:r{행번호}` |
| `rank_number` | `string` | 순위번호. 주등기: `"1"`, 부기: `"1-1"` |
| `previous_rank` | `string \| null` | 종전 순위번호. 전산이기 시 `"(전 3)"` → `"3"` |
| `purpose` | `object` | [PurposeInfo](#purposeinfo) — 등기목적 |
| `receipt` | `object` | [ReceiptInfo](#receiptinfo) — 접수 정보 |
| `cause` | `object` | [CauseInfo](#causeinfo) — 등기원인 |
| `content` | `object` | [EntryContent](#entrycontent) — 권리자및기타사항 |
| `status` | `object` | [EntryStatus](#entrystatus) — 말소·변경 상태 |
| `provenance` | `object` | [Provenance](#provenance) — 출처 추적 |
| `raw_text` | `string` | 원본 셀 텍스트 |
| `confidence` | `float` | 파싱 신뢰도 (0.0 ~ 1.0) |

## PurposeInfo

등기목적 컬럼에서 추출됩니다. 원문 보존과 함께 정규화·분류 정보를 제공합니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `raw` | `string` | PDF 원문 (예: `"소유권이전"`) |
| `normalized` | `string` | 공백 제거 정규화 |
| `right_type` | `string` | 권리종류 분류키 (예: `"ownership_transfer"`, `"mortgage"`, `"lease"`) |
| `action_type` | `string` | 액션 유형 (예: `"create"`, `"transfer"`, `"cancel"`) |
| `target_rank` | `string \| null` | 대상 순위번호 (부기·말소 시) |
| `target_type` | `string \| null` | 대상 권리종류 (을구 부기 시) |
| `target_owner` | `string \| null` | 대상 소유자명 (갑구 부기 시) |
| `sub_purposes` | `array` | 복합 목적 시 하위 [PurposeInfo](#purposeinfo) 목록 |

### right_type 분류

| right_type | 설명 | 예시 |
|------------|------|------|
| `ownership_transfer` | 소유권이전 | 소유권이전, 소유권일부이전 |
| `ownership_preservation` | 소유권보존 | 소유권보존 |
| `mortgage` | 근저당권/저당권 | 근저당권설정, 근저당권이전, 저당권설정 |
| `lease` | 임차권 | 임차권등기, 임차권등기명령, 임차권설정 |
| `jeonse` | 전세권 | 전세권설정 |
| `superficies` | 지상권 | 지상권설정 |
| `easement` | 지역권 | 지역권설정 |
| `provisional_registration` | 가등기 | 소유권이전청구권가등기 |
| `seizure` | 압류 | 압류 |
| `provisional_seizure` | 가압류 | 가압류 |
| `auction` | 경매 | 임의경매개시결정, 강제경매개시결정 |
| `trust` | 신탁 | 신탁 |
| `injunction` | 가처분 | 처분금지가처분 |
| `owner_change` | 등기명의인표시변경 | 등기명의인표시변경, 등기명의인주소변경 |
| `supplementary` | 부기등기 | 부기등기 |
| `generic` | 기타 | 분류 불가 항목 |

### action_type 분류

| action_type | 설명 |
|-------------|------|
| `create` | 신규 설정 (소유권보존, 근저당권설정 등) |
| `transfer` | 이전 (소유권이전, 근저당권이전 등) |
| `partial_transfer` | 일부이전 (소유권일부이전, 근저당권일부이전 등) |
| `change` | 변경 (근저당권변경 등) |
| `correct` | 경정 (등기명의인표시경정 등) |
| `cancel` | 말소 (근저당권설정등기말소 등) |
| `annotate` | 부기 (부기등기, 건물만에관한것임 등) |

<Accordion title="말소 부기등기 예시">
  순위번호 5번의 경매등기를 말소하는 경우:
  ```json
  {
    "purpose": {
      "raw": "5번 임의경매개시결정등기말소",
      "normalized": "5번임의경매개시결정등기말소",
      "right_type": "auction",
      "action_type": "cancel",
      "target_rank": "5",
      "sub_purposes": []
    }
  }
  ```
</Accordion>

<Accordion title="복합말소 예시">
  1번 근저당권과 2번 지상권을 동시에 말소:
  ```json
  {
    "purpose": {
      "raw": "1번근저당권설정, 2번지상권설정등기말소",
      "normalized": "1번근저당권설정,2번지상권설정등기말소",
      "right_type": "mortgage",
      "action_type": "cancel",
      "target_rank": "1",
      "sub_purposes": [
        {
          "raw": "1번근저당권설정등기말소",
          "right_type": "mortgage",
          "action_type": "cancel",
          "target_rank": "1"
        },
        {
          "raw": "2번지상권설정등기말소",
          "right_type": "superficies",
          "action_type": "cancel",
          "target_rank": "2"
        }
      ]
    }
  }
  ```
</Accordion>

## EntryContent

권리자및기타사항 컬럼의 구조화된 데이터입니다. 권리종류에 따라 해당 필드만 값이 설정됩니다.

## 매매목록

토지 문서에서 복수 필지를 한 거래로 처리할 때 사용됩니다. 건물·집합건물에서는 보통 빈 배열입니다.

```json
{
  "list_number": "2016-553",
  "trade_amount": 85000000,
  "items": [
    {
      "serial_number": "1",
      "property_description": "[토지] 경상북도 문경시 농암면 내서리 733",
      "rank_number": "8",
      "registration_cause": "매매",
      "correction_cause": ""
    }
  ]
}
```

갑구 항목의 `content.trade_list_number`와 `trade_lists[].list_number`로 연결됩니다.

## 버전 호환성

### v1.4.0 → v1.5.0 주요 변경

| 항목 | v1.4.0 | v1.5.0 |
|------|--------|--------|
| 갑구/을구 모델 | `SectionAEntry` / `SectionBEntry` 분리 | `RegistryEntry` 통합 |
| 등기목적 | `purposes: List[RegistrationPurpose]` | `purpose: PurposeInfo` (단일 객체, `sub_purposes`로 복합 처리) |
| 인물 정보 | `owners`, `creditors`, `debtors`, `mortgagees` 분산 | `persons[]` + `role` 통합 |
| 금액 정보 | `max_claim_amount`, `deposit_amount` 등 분산 | `amounts[]` + `type` 통합 |
| 최상위 구조 | `RegistryData` | `RegistryDocument` (meta, profile, source, diagnostics 추가) |
| 출처 추적 | 없음 | `provenance` (page_no, bbox, merged_from) |
| 항목 식별 | 없음 | `entry_id` (예: `"gap:p2:r5"`) |
| 항목 신뢰도 | 없음 | `confidence` (0.0 ~ 1.0) |

<Note>
  v1.4.0 응답을 받는 경우 `purposes` (배열)와 `details` (갑구/을구별 상이) 필드를 사용합니다.
  정확한 v1.4.0 스키마가 필요하면 별도 문의해 주세요.
</Note>
